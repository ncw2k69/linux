## Sudo
Install `sudo` as  root
```
apt install sudo
```
Add user in list
```
nano /etc/sudoers.d/me
```
Content
```
your-user-name-here     ALL=(ALL)       ALL
```
## APT
Open
```
sudo nano /etc/apt/apt.conf
```
Add
```
APT::Install-Recommends "false";
APT::Install-Suggests "false";
```

## Fixing HALT on Power Off or Reboot
There is an issue related to the HSUART DMA support in the kernel that will hang on reboot or shutdown. 
Open `blacklist.conf`
```
sudo nano /etc/modprobe.d/blacklist.conf
```
Add these lines
```
blacklist dw_dmac_core
install dw_dmac /bin/true
install dw_dmac_core /bin/true
```
Then, update `initramfs`
```
sudo update-initramfs -u
```
_Source_: [Debian Wiki - Installing Debian On Dell Wyse 3040](https://wiki.debian.org/InstallingDebianOn/Dell/Wyse%203040#System_Summary)
## Wake on Lan
### Bios
Enable it in Bios - Power Management and set `Wake on LAN` to `LAN Only` or `LAN with PXE Boot`.
### OS Level
Check if option is available
```
sudo ethtool enp1s0 | grep Wake
```
If you see `Wake-on: d` option is available, but is disable.

To enable it one-time run:
```
sudo ethtool -s enp1s0 wol g
```
To make it permanent, add the wake option in interfaces
```
sudo nano /etc/network/interfaces
```
After `iface enp1s0 inet dhcp`, on next row, add a `tab` ident and add `ethernet-wol g`, must look like this
```
allow-hotplug enp1s0
iface enp1s0 inet dhcp
      ethernet-wol g 
```
Then reboot
```
sudo reboot
```
Check if option got activated
```
sudo ethtool enp1s0 | grep Wake
```
If you see `Wake-on: g` option is active.

To test it, power it off
```
sudo poweroff
```
Send magic wakeup package from any Device in that lan network
```
wakeonlan XX:XX:XX:XX:XX:XX
```
The `3040` wyse should wake up. In case is not working, recheck all the prev steps

_Source_: [Debian Wiki - WakeOnLan](https://wiki.debian.org/WakeOnLan#Checking_WOL)

## Disk Check
Only for test env, we don't care about data, interested in fast startups

Open `fstab`
```
sudo nano /etc/fstab
```
Change column `pass` value from `1` to `0`, skips disk checks
```
# <file system>                    <mount point>   <type>  <options>          <dump>  <pass>
UUID=xxxx-xxxx-xxxx-xxxx-xxxxxxx   /               ext4    errors=remount-ro  0       1
to
UUID=xxxx-xxxx-xxxx-xxxx-xxxxxxx   /               ext4    errors=remount-ro  0       0
```


## GRUB Timeout
Open the `grub` file:
```
sudo nano /etc/default/grub
```
Change `GRUB_TIMEOUT` to what you want `0` or `1`... as high or low as you want
```
GRUB_TIMEOUT=5
to
GRUB_TIMEOUT=1
```
Update grub
```
sudo update-grub
```

## HTOP CPU Freq & Temp
Open htop config file
```
nano ~/.config/htop/htoprc
```
Update `show_cpu_frequency` and `show_cpu_temperature` to `1`
```
...
show_cpu_frequency=1
show_cpu_temperature=1
...
```
Optional - Copy same config to ROOT/SUDO 
```
sudo cp -r ~/.config/htop/ /root/.config/
```
